# Task Packet

## Objective

Freeze the Phase 14.5 external-runtime launcher safety contract before any
launcher implementation or supervised runtime invocation.

## Context

`phase14-runtime-adapter-01` is accepted and integrated on `main`. The
operator selected OpenCode for the first pilot; its bounded `--version` probe
reported `available`. The planned pilot provenance is worker
`external-agent-platform-33`, this coordination repository, reviewer
`ORCHESTRATOR`, branch
`agent/external-agent-platform-33/phase14.5-pilot-01`, worktree
`worktrees/phase14.5-pilot-01`, and rollback owner `ORCHESTRATOR`.

## Constraints

- Documentation and test-planning only: do not implement a launcher, create a
  scheduler, create a run manifest, dispatch a task, or launch OpenCode.
- Treat the planned pilot provenance as an allowlisted contract input, not as
  permission to execute a runtime.
- The future launcher must default to dry run and fail closed for every
  missing, malformed, mismatched, disabled, timed-out, or non-zero condition.
- Do not record credentials, prompts, task bodies, source content, executable
  paths, runtime output, or session transcripts in audit fields or examples.

## Implementation Notes

Write one concise operator-facing contract document containing the normative
rules, a threat/risk table, audit/redaction requirements, and a deterministic
reject-path test matrix. Record the durable pilot-selection decision in
`DECISIONS.md`. Later implementation work must consume this contract rather
than infer commands from task text or environment variables.

## Token And Resource Impact (If Applicable)

Measured baseline: this specification task makes no model call, polling loop,
runtime launch, manifest write, or worker-state mutation. The contract requires
future dry-run-first behavior, bounded one-shot execution, and redacted audit
records; disable behavior is a mandatory reject path.

## Validation Steps

Run `python scripts/validate_coordination_files.py`, inspect the contract for
all required safety fields and prohibited logging content, and run
`git diff --check`.

## Escalation Rules

Stop and open an incident if the contract requires a runtime launch, arbitrary
shell input, credentials, task lifecycle mutation, a global resource change,
or a scope change outside the task card. Do not proceed to Phase 14.5-03 or a
supervised pilot without separate review and operator authorization.
