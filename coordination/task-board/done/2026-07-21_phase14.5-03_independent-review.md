## Review Standard And Required Checks

1. **Dry-run-only safety:** verify `launcher_dry_run.py` neither imports nor
   calls `subprocess`, `os.system`, `Popen`, shell execution, or any runtime
   invocation; it must not create manifests, inboxes, worktrees, scheduler
   state, or task-lifecycle state.
2. **Safe observability:** verify all returned and CLI-emitted fields are
   limited to safe IDs, decision categories, and booleans. Paths, argv, task
   content, prompts, credentials, runtime output, and error output must not
   be emitted.
3. **Fail-closed contract coverage:** inspect both implementation and tests
   for deterministic handling of missing manifest, invalid manifest, disabled
   state, missing or malformed inbox, wrong worker/project/task, wrong
   branch/worktree, unallowlisted runtime/executable, absent one-shot approval,
   timeout, non-zero exit, credential prompt, and monitor anomaly. Missing
   executable coverage is a finding, not an assumption.
4. **Determinism and boundary behavior:** valid bound input must yield only
   `dry_run_ready`; invalid input must yield the contract-defined safe
   `deny_*` or `stopped_*` result with no fallback or retry.
5. **Interface and documentation:** confirm `launcher-dry-run` is exposed by
   `scripts/orchestrate.py` and the guide accurately states its dry-run-only,
   non-approval boundary.
6. **Scope and evidence:** verify changed files match the original task card's
   allowed scope, the delivery report is accurate, and the submitted test,
   compilation, coordination validation, and `git diff --check` evidence is
   reproducible.
