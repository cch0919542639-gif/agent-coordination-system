# Task Packet

## Objective

Implement the manifest, approval, provenance, timeout, and safe-audit
validation foundation from the accepted Phase 14.5-04 design. This task must
remain incapable of launching an external runtime.

## Context

The frozen source and accepted design are:

- `docs/operations/phase14.5-launcher-safety-contract.md`
- `docs/operations/phase14.5-supervised-launch-design.md`
- `coordination/reviews/review-phase14.5-04.md`

The single bound pilot is OpenCode with worker
`external-agent-platform-33`, reviewer/rollback owner `ORCHESTRATOR`, branch
`agent/external-agent-platform-33/phase14.5-pilot-01`, and worktree reference
`worktrees/phase14.5-pilot-01`.

## Constraints

- Never import or call `subprocess`, `os.system`, `Popen`, shell execution, or
  any process API.
- Never define a real argv, executable path, command string, or runtime
  invocation.
- Never create, sign, mutate, or consume a real manifest, inbox, approval,
  worktree, scheduler record, or task lifecycle state.
- Treat `supervised_one_shot` as ineligible for execution in this task even
  when schema validation succeeds; emit a safe decision requiring a later
  separately approved launch-executor task.
- Emit or store only safe IDs, categories, booleans, numeric timeout bounds,
  and digests. Never log credentials, prompts, task bodies, argv, paths, or
  runtime output.

## Implementation Notes

Use standard-library JSON and pure functions. Prefer a new focused validation
module over changing runtime-adapter behavior. The module should validate only
in-memory JSON-shaped data and return deterministic safe decision records.
Tests must exercise missing/malformed/mismatched manifest and approval input,
disabled state, unallowlisted identity, timeout, unsafe audit schema, and the
no-execution boundary.

## Token And Resource Impact (If Applicable)

Not applicable. This is deterministic local validation code with no model call,
polling, external network action, or runtime launch.

## Validation Steps

- Run focused unit tests for the new module and existing dry-run safety tests.
- Run Python compilation, `python scripts/validate_coordination_files.py`, and
  `git diff --check`.
- Provide test results and a source-level no-process verification in the
  delivery report.

## Escalation Rules

Stop and open an incident if implementation needs a real command, executable
path, process creation, mutable operational artifact, broader pilot identity,
or a change outside the allowed scope.
