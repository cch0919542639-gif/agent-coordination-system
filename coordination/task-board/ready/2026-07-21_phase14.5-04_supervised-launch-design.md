# Task Packet

## Objective

Design, but do not implement or execute, the supervised one-shot launch path
that may follow the accepted dry-run-only preflight.

## Context

Use `docs/operations/phase14.5-launcher-safety-contract.md` as the frozen
source. The only prospective pilot remains `opencode` with worker
`external-agent-platform-33`, reviewer/rollback owner `ORCHESTRATOR`, bound
branch `agent/external-agent-platform-33/phase14.5-pilot-01`, and bound
worktree `worktrees/phase14.5-pilot-01`.

## Constraints

- This is design-only: never invoke OpenCode, any subprocess, shell command,
  runtime probe, or external service.
- Do not create, sign, write, or consume a manifest, inbox, worktree, run
  record, approval record, or task lifecycle change.
- Do not define a real argv or executable path. A future implementation needs
  a separately approved immutable manifest-bound argv design.
- Do not broaden to another runtime, worker, project, branch, worktree, or
  automatic retry/approval flow.
- Keep all audit examples to allowlisted safe fields; never include credentials,
  account IDs, prompts, task bodies, paths, argv, or runtime output.

## Implementation Notes

Create `docs/operations/phase14.5-supervised-launch-design.md` covering:

1. immutable manifest and approval-reference schemas with validation rules;
2. dry-run versus supervised-one-shot state transitions and terminal stops;
3. exact authorization, expiry, global-disable, stop-request, timeout, and
   process-group requirements for a later implementation;
4. safe audit-record schema and redaction rules;
5. deterministic positive and reject-path test matrix; and
6. explicit non-goals and the user approval needed before any implementation
   task may be opened.

## Token And Resource Impact (If Applicable)

Not applicable. This is bounded documentation design; it adds no runtime,
polling loop, model call, or external network action.

## Validation Steps

- Cross-check every design rule against the frozen Phase 14.5 contract.
- Confirm the design contains no executable argv, real path, credential,
  runtime output, or launch instruction.
- Run `python scripts/validate_coordination_files.py` and `git diff --check`.

## Escalation Rules

Stop and open an incident if the design requires a launch, a mutable artifact,
a real command, an unapproved pilot identity, or any change outside the
allowed documentation and coordination scope.
