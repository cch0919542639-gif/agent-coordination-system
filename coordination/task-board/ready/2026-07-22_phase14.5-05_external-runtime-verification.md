2. Verify the single pilot bindings, manifest and approval validation, timeout
   bound, audit allowlist, disabled state, and reject categories match the

   the future-executor decision.
4. Verify a non-allowlisted runtime returns `deny_allowlist` and supervised
   requests can only yield a safe future-executor-required decision, never an
   execution decision.
5. Run focused tests with the configured Python runtime. If `pytest` is absent,
   record that limitation and use an equivalent standard-library harness; do
   not install dependencies.
6. Run Python compilation, coordination validation, and `git diff --check`.
